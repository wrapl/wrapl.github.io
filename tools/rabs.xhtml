Wrapl, The Programming Language

Rabs

Rabs is the build tool used to build all the entire Wrapl system. Although originally created to build Wrapl, Rabs was designed to be a completely general purpose build tool.

Features

  • Language Agnostic: Rabs was designed from the ground up to be completely agnostic to any programming language or use case. There are no built-in functions specific to any language.
  • Supports Large Projects: each directory in a project can have its own build script, with settings inherited from parent directories.
  • Fully scriptable: Rabs provides a complete imperative language called Minilang, which is be used to configure the building of the project as well as for the build functions themselves.

Command Line Options

Within a folder containing a _minibuild_ file (see the guide for more details), running rabs will load _minibuild_ in that folder and build the DEFAULT target. rabs accepts a number of command line options for controlling the build.

OptionDescription
-cPrints commands as they are executed.
-sPrints targets and counts as they as built.
-p nUses n threads in parallel.
-D key=valueSets key/value pair.
-w(EXPERIMENTAL) After building, rabs will listen for file changes and rebuild affected targets.
-vPrint version number and exit.
